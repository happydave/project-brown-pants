# Material attribution

PBR material sets in this directory are baked outputs from the **asset-harness**
`pbr-materials` track (generated locally; albedo via Z-Image, maps derived locally). The
workflow/seed/prompt is the source of truth in the asset-harness repo; these PNGs are the
cached, versioned output.

License: Apache-2.0 / MIT (per the generating models' terms recorded in the asset-harness
track docs).

| Set | Maps | Used by |
| --- | ---- | ------- |
| `hull_panel` | albedo, normal, metallic_roughness, occlusion | shared / fallback craft skin (`-- skins`), WI 582/583 |
| `rocky_ground` | albedo, normal, metallic_roughness, occlusion | textured ground (`-- skins`/`-- play`/`-- autopilot`/`-- launch`), WI 588 |
| `aluminium_panel` | albedo, normal, metallic_roughness, occlusion | structural material **aluminium** (brushed aluminium panel), WI 624 |
| `steel_plate` | albedo, normal, metallic_roughness, occlusion | structural material **steel** (polished steel plate), WI 624 |
| `titanium_panel` | albedo, normal, metallic_roughness, occlusion | structural material **titanium** (satin titanium alloy), WI 624 |
| `carbon_weave` | albedo, normal, metallic_roughness, occlusion | structural material **composite** (carbon-fibre weave), WI 624 |

The four structural sets (WI 624) were generated by the `pbr-materials` track driver
`prototypes/gen_material_library.py` (seed 424242, 1024², albedo via Z-Image Turbo on `ai2`, maps
derived locally with `derive_pbr.py`). Bonus planetary-surface sets (`sand_dune`, `ice_sheet`,
`basalt_rock`) from the same run are banked in the asset-harness library, not yet imported here.
